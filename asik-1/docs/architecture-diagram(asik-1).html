Business Rules & Architecture Decisions

RuleWhere EnforcedDetail
Amount stored as int64 DOMAIN All amounts in cents. BIGINT in PostgreSQL. No float64 anywhere.
Amount must be > 0 USE CASE Validated in Order.Validate() and Gin binding. Returns 400.
Only Pending orders can be cancelled DOMAIN Order.CanBeCancelled() — returns 409 Conflict if Paid.
Amount > 100,000 cents → Declined USE CASE Payment limit enforced in PaymentUseCase.Authorize().
Payment timeout = 2 seconds DELIVERY context.WithTimeout(2s) in OrderHandler.CreateOrder(). Returns 503.
Database per service Infrastructure orders_db and payments_db are entirely separate Postgres instances.
No shared code / models Architecture Each service has its own internal/domain. No common package.
Idempotency (Bonus) USE CASE Idempotency-Key header. UNIQUE constraint in DB. Safe retries.
Composition Root / Manual DI main.go All dependencies wired by hand in main.go. No DI framework.
